React

React 是什么

React 是一个用于构建可组合的用户界面的库。它鼓励创建可重用的UI组件，呈现随时间改变的数据。
React不使用模板
传统上，Web应用程序UI是使用模板或HTML指令构建的。这些模板指定了使用来构建UI的完整的抽象集。
React通过将用户界面分成组件来以不同的方式构建用户界面。这意味着React使用一个真正的，全功能的编程语言来渲染视图，我们认为这是一个优于模板的优点，原因如下：
  • JavaScript是一种灵活，强大的编程语言，具有构建抽象的能力。这在大型应用中非常重要。
  • 通过将其标记与其相应的视图逻辑统一起来，React实际上可以使视图更容易扩展和维护。
  • 通过将对标记和内容的理解烘焙到JavaScript中，没有手动字符串连接，因此XSS漏洞的表面积更小。
当组件首次初始化时，调用render方法，生成视图的轻量表示。从该表示，产生一串标记，并注入到文档中。当数据更改时，将再次调用render方法。为了尽可能有效地执行更新，我们使用新的调用来更新上一次调用的返回值，并生成一组最小的更改以应用于DOM。
从render返回的数据既不是字符串也不是DOM节点 - 它是DOM应该看起来像什么的轻量级描述。
React认为一个组件应该具有如下特征：
  • 可组合（Composeable）：一个组件易于和其它组件一起使用，或者嵌套在另一个组件内部。如果一个组件内部创建了另一个组件，那么说父组件拥有（own）它创建的子组件，通过这个特性，一个复杂的UI可以拆分成多个简单的UI组件；
  • 可重用（Reusable）：每个组件都是具有独立功能的，它可以被使用在多个UI场景；
  • 可维护（Maintainable）：每个小的组件仅仅包含自身的逻辑，更容易被理解和维护；
  • 可测试（Testable）：因为每个组件都是独立的，那么对于各个组件分别测试显然要比对于整个UI进行测试容易的多。
虚拟DOM:
React是一个全新思路的前端UI框架，它完全接管了UI开发中最为复杂的局部更新部分，擅长在在复杂场景下保证高性能；同时，它引入了基于组件的开发思想，从另一个角度来重新审视UI的构成。通过这种方法，不仅能够提高开发效率，而且可以让代码更容易理解，维护和测试。

第一种：sort()方法
如果调用该方法时没有使用参数，将按字母顺序对数组中的元素进行排序，说得更精确点，是按照字符编码的
顺序进行排序。要实现这一点，首先应把数组的元素都转换成字符串（如有必要），以便进行比较。
如果想按照其他标准进行排序，就需要提供比较函数，该函数要比较两个值，然后返回一个用于说明这两个值的
相对顺序的数字。比较函数应该具有两个参数 a 和 b，其返回值如下：
若 a 小于 b，在排序后的数组中 a 应该出现在 b 之前，则返回一个小于 0 的值。
若 a 等于 b，则返回 0。
若 a 大于 b，则返回一个大于 0 的值。

什么是HTML本地存储？

具有本地存储，web应用程序可以在用户的​​浏览器中本地存储数据。
HTML5之前，应用程序数据必须被存储在cookie中，包括在每一个服务器请求。本地存储更安全，以及大量的数
据可以在本地存储，不影响网站性能。不像cookie，存储限制是大得多（至少5MB）和信息不会传送到服务器。
本地存储是每原点（每个域和协议）。所有网页，从一个起点，可以存储和访问相同的数据。

HTML本地存储对象

HTML本地存储提供了用于存储在客户机上的数据的两个对象：
1. window.localStorage - 没有过期日期存储数据
2. window.sessionStorage - 一个会话存储数据（当浏览器标签页关闭数据丢失）
使用本地存储之前，请检查localStorage的和sessionStorage的浏览器支持
在localStorage的对象存储与没有到期日的数据。当浏览器被关闭的数据不会被删除.名称/值对总是存储为字符串。记得在需要时将它们转换为另一种格式！
sessionStorage对象等于localStorage的对象，除了它存储的数据为仅一个会话。当用户关闭浏览器特定选项卡中的数据被删除。
